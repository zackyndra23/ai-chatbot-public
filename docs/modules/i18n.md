# Module — `i18n`

Centralized translation loader for the chatbot. Replaces 13 scattered per-language palette dicts across `system_detection`, `service_agent`, and `meeting_arrangement` modules.

Introduced 2026-05-13 as foundation for the OOC Response Engine — see `docs/superpowers/specs/2026-05-13-ooc-response-engine-design.md` §4 for the full design rationale.

## Purpose

Single API (`t(key, lang, **fmt_kwargs)`) for looking up user-facing strings. Backs:
- OOC engine response templates (cold-start + mid-flow shapes)
- Abandonment acknowledgments + trigger phrases
- Greeting palette (post-migration)
- Service-agent opener palette + banned-form lint (post-migration)
- Rescue soft-bridge phrases (post-migration)
- Picker labels (post-migration)
- Meeting footer / picker preamble (post-migration)

Validates at chatbot startup. CRITICAL validation failures refuse to start; WARN/INFO log and continue.

## Public API

```python
from modules.i18n import t, service_label, field_label, ui_term, validate_all

# Normal string lookup with placeholder substitution:
text = t("ooc.OOC-PARTNERSHIP.cold_start", "id",
         partnership_url="https://...",
         indo_email="contact@...",
         indo_phone="+62...",
         business_hours="Mon-Fri 09:00-18:00 WIB")
# Returns str.

# List-typed entries (palettes, trigger-phrase banks) return list[str]:
palette = t("greeting.palette", "en")  # returns ["Hello!", "Hi!", ...]

# Glossary lookups (Task 5 — from glossary.yaml):
service_label("wbs", "id")  # → "Sistem Whistleblowing (WBS)"
field_label("case_handler_quantity", "en")  # → "Number of Case Handlers"
ui_term("qualification", "id")  # → "kualifikasi"
# All three fall back to English when lang is missing from glossary.

# Run validation at startup or in CI:
report = validate_all()
report.raise_if_critical()  # raises MissingKeyError if any CRITICAL issue
```

## YAML formats

### `schema.yaml` — authoritative key catalog

```yaml
ooc.OOC-PARTNERSHIP.cold_start:
  surface: "ooc"
  category: "OOC-PARTNERSHIP"
  shape: "cold_start"
  placeholders: [partnership_url, indo_email, indo_phone, business_hours]
  max_words: 200
  tone: "professional_courteous"
  required_for: [en, id]   # Langs that MUST have this key at startup
  auto_bidi_wrap_extracted_vars: false
  template_variant_for_lang: {}
  field_label_lang_exceptions: []
  notes: "Cold-start partnership inquiry — Partnership URL + Indo email/phone routing."
```

List-typed entries declare `type: "list"`:

```yaml
greeting.palette:
  surface: "greeting"
  placeholders: []
  type: "list"
  required_for: [en, id]
```

### `strings/{lang}.yaml` — per-lang translations

```yaml
ooc.OOC-PARTNERSHIP.cold_start:
  text: |
    Thanks for considering a partnership with Integrity — ...
  status: verified              # 6-state enum below
  reviewer: "@design"
  reviewed_date: "2026-05-13"
  ai_assisted: false
  baseline_hash: "a3f2c..."     # sha256 of en.yaml text at verification time
```

List-typed entries use a YAML list under `text`:

```yaml
greeting.palette:
  text:
    - "Hello! How can I help you today?"
    - "Hi! What brings you here today?"
  status: verified
```

## 6-state status enum

| Status | Render behavior | Audit log |
|---|---|---|
| `verified` | Render text directly | (none) |
| `draft` | Render text directly | `i18n_draft_translation_used` |
| `needs_revision` | Render text directly (graceful degrade) | `i18n_needs_revision_translation_used` |
| `missing` | Fall back to English + log RUNTIME_FALLBACK | drives prioritization |
| `stale_re_review` | Render text + log INFO | auto-detected via baseline_hash mismatch |
| `deprecated` | Fall back to English + log `usage_of_deprecated_key` | alerts consumer code drift |

## 4-tier validation severity

| Severity | Examples | Action |
|---|---|---|
| `CRITICAL` | required key absent from required-lang YAML; placeholder mismatch between schema and text | **Refuse to start** (raises `MissingKeyError`) |
| `WARN` | required key present with `status: missing` at startup; banned-form lint hit | Log + continue |
| `INFO` | `baseline_hash` mismatch on `verified` entry | Auto-flip to `stale_re_review`; log |
| `RUNTIME_FALLBACK` | `t()` called for a key whose status is `missing` or `deprecated` in target lang | Render English + log; NEVER crashes request |

## Runtime fallback chain (the `t()` call)

```
t(key, lang, **kwargs)
    │
    ├── Look up entries[(key, lang)]
    │       └── If missing OR status ∈ {missing, deprecated}:
    │             log RUNTIME_FALLBACK
    │             fall back to entries[(key, "en")]
    │
    ├── If still missing in en: raise MissingKeyError
    │
    └── entry.text.format(**kwargs) → return rendered string
        (or return entry.text as-is when it's a list)
```

## Adding a new key

1. Add the key + schema metadata to `schema.yaml`, including `required_for`.
2. Add the English baseline to `strings/en.yaml` with `status: verified`.
3. For each non-en lang in `required_for`: add the translation with `status: draft` (or `status: missing` + empty text if waiting on translator).
4. Run `pytest tests/test_i18n_loader.py` + `python -c "from modules.i18n import validate_all; r = validate_all(); print(r)"` to confirm no CRITICAL issues.

## Adding a new language

1. Create `strings/{lang}.yaml` with all schema keys that include the new lang in `required_for`.
2. Drafted translations land with `status: draft`; SME reviews then flip to `verified`.
3. For phased rollout: leave non-Phase keys with `status: missing` so runtime falls back to English.
4. The 17-language canonical set per spec §Constraint #7: `id, ms, en, fr, de, it, pt, es, vi, th, da, zh, ja, ru, ko, tl, ar`.

## Test patterns

`tests/test_i18n_loader.py` uses `tmp_path` fixtures to build per-test isolated YAML directories. Tests never touch the production `modules/i18n/` files — keeps loader logic decoupled from translation content.

Tests cover:
- Verified translation with placeholder substitution (LC-1)
- Runtime fallback to English when entry status=missing/deprecated (LC-4)
- CRITICAL on missing required key or placeholder mismatch (LC-5)
- WARN on required-key with status=missing at startup
- INFO on baseline_hash drift → auto-flip to stale_re_review
- List-typed entries returned as list
- Empty base_dir loads empty registry (smoke)

## Gotchas

- `extra={...}` in `log.warning(...)` cannot use keys that collide with `LogRecord` reserved names (`message`, `asctime`, etc.). This module uses `detail` instead.
- YAML files MUST be UTF-8 (Windows codec gotcha — `open(..., encoding='utf-8')` is explicit in loader).
- The registry is a process-level singleton. Tests that need a fresh registry call `_reset_registry_for_tests()`.
- Pydantic v2 is in use (project uses `pydantic.BaseModel`, NOT `langchain_core.pydantic_v1`).
